# Metadata

This directory contains language metadata for the FLEURS-R dataset.

## Files

- `languages.json`: Main metadata file mapping FLEURS directory names to
  language info (full name, glottocode, speaker counts, IECOR name, taxonset).
  Generated by `pipeline/download_fleurs_metadata.py`.
- `glottolog.csv`: Glottolog language data (coordinates, lineage codes)
- `n_speakers.csv`: Speaker population data from multiple sources
- `phoible.csv`: Phoible feature set for FLEURS-R languages
- `squim.csv`: Squim objective speech quality assessment metrics on audio files
- `wikimedia_speakers.json`: Raw speaker data extracted from Wikipedia infoboxes

## languages.json

The `full` key was retrieved from Glottolog using the glottocode. Speaker counts
come from two sources:

- `wikimedia`: Wikipedia infoboxes (often sourced from Ethnologue)
- `linguameta`: Google's LinguaMeta dataset

The `iecor` key maps to IE-CoR language names for Indo-European languages. The
`taxonset` key indicates BEAST2 calibration group membership (Germanic, Slavic,
Indo-Aryan, Iranian, Romance, Celtic, Baltic).

## Known data issues

- **af_za (Afrikaans)**: dev and test splits have more `.wav` files on disk (416
  dev, 414 test) than entries in the corresponding TSVs (197 dev, 264 test). The
  369 orphan audio files have no transcript entry and no sentence index. All
  other languages have exact parity between audio files and TSV entries.
